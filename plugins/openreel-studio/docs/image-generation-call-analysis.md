# 图片生成调用链六维审计

目标场景：用户在 Codex 中要求生成一张图片并直接放到当前 OpenReel 画布，同时把生成后端交给 Codex 选择。默认后端采用 Codex 内置图片生成。

## 稳定调用链

1. 当前 Codex 会话首次操作 OpenReel 时完成连接和项目选择，后续图片复用该状态。
2. Codex 内置图片生成执行 1 次并产出最终本地文件。
3. `openreel_publish_generated_image` 执行 1 次。
4. 新版 OpenReel 使用 1 个 multipart 请求完成节点创建、图片保存、产物登记和完整节点事件广播。
5. 发布结果中的 completed 节点作为本次操作的持久化验证。

## 六个维度

| 维度 | 设计依据 | 稳定目标 |
| --- | --- | --- |
| 1. 模型生成 | 一张最终图片对应一个明确生成后端。 | Codex 图片生成 1 次，OpenReel provider 调用 0 次。 |
| 2. Codex 工具 | 发布工具同时创建节点并写入成品。 | `imagegen` + publish，共 2 次宿主工具调用。 |
| 3. 插件到 OpenReel HTTP | 服务端导入动作合并节点与媒体持久化。 | 新版服务 1 次 import-image；兼容服务使用 create + upload。 |
| 4. 持久化与一致性 | 画布直接接收包含输出的 completed 节点。 | 成功返回完整节点；兼容失败路径按节点 id 继续上传。 |
| 5. 画布刷新 | `snapshot_complete` 事件携带完整节点内容。 | 前端一次合并标题、字段、状态、媒体和历史。 |
| 6. 失败与重试 | 生成文件与画布节点身份可以独立复用。 | 生成阶段重试生成；发布阶段复用原文件和已返回节点 id。 |

## 调用次数预算

以下预算省略每个会话只执行一次的连接、列表和选择：

- 正常单图：1 次 Codex 图片生成 + 1 次插件 publish + 1 次 OpenReel 导入 HTTP + 1 次完整节点事件。
- 显式 OpenReel provider：1 次动态合同预检 + 1 次 create 插件调用 + 1 次 run 插件调用；create 负责字段持久化与合同校验，run 负责触发模型费用和媒体任务。
- 兼容回退：首次能力探测后缓存服务端支持状态；当前图片复用同一文件执行 create + upload，后续图片直接采用已确认路径。
- 结果读取：publish 的完整 completed 节点完成常规验证；返回内容缺失或后续步骤依赖精确拓扑时读取节点或画布。

插件对服务端 UUID 直接传递。等待和上传拿到内部 UUID 时直接执行；可见编号（如 `#3`）在调用边界解析一次，确保定位到持久化节点。

## 后端选择

- 用户把后端交给 Codex：使用 Codex 内置图片生成。
- 用户明确选择 OpenReel 模型、provider 或当前模型配置：读取动态节点合同后创建并运行 OpenReel 节点。
- 用户要求比较两种结果：两个后端各生成一次，并分别发布为独立节点。
